Developed a complete MLOps pipeline simulating a fintech loan default system. Implemented DVC-style data versioning, drift detection, and schema validation to prevent data-related failures. Integrated MLflow for experiment tracking, logging parameters, metrics, and models, enabling reproducible training and automated best model selection using F1 score.

Key Learnings
Data versioning is critical for debugging ML failures
Silent data drift can break models even if code doesn’t change
Schema validation prevents production issues early
MLflow ensures experiment reproducibility and transparency
F1 score is better than accuracy for imbalanced datasets
